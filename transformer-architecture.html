Step 1 & 2

Token Embeddings + Positional Encoding

Token Embeddings

Each word/subword is mapped to a learned dense vector (e.g., 512 or 768 dimensions).

"The" → [0.21, -0.5, 0.8, ...] "cat" → [0.73, 0.12, -0.3, ...] "sat" → [-0.1, 0.44, 0.9, ...]

Positional Encoding

Since Transformers process all words in parallel, we must tell the model where each word is.

// Sinusoidal pattern: PE(pos, 2i) = sin(pos / 10000^(2i/d)) PE(pos, 2i+1) = cos(pos / 10000^(2i/d)) // Final input: input = embedding + positional_encoding

Parallel Perspectives

Multi-Head Attention

Instead of one attention computation, run multiple in parallel. Each head learns to focus on different relationships.

H1Grammar
H2Meaning
H3Position
H4Syntax
H5Coreference
H6Negation
H7Entities
H8Relations
MultiHead = Concat(head1, head2, ..., headh) · WO
Why multiple heads?

One head might learn "cat sat" (subject-verb), another "sat on mat" (action-location). Together, they capture richer relationships.

Processing & Stability

Feed-Forward + Residual Connections

Feed-Forward Network

After attention gathers information, the FFN processes it. Applied to each position independently.

FFN(x) = ReLU(x·W1 + b1)·W2 + b2

Typically expands dimension 4x (e.g., 512 → 2048 → 512). This is where the model stores "knowledge."

Add & Normalize

Two critical tricks that make deep Transformers trainable:

// Residual Connection output = x + SubLayer(x) // Layer Normalization output = LayerNorm(output)

Residual: Info can skip layers (prevents vanishing gradients)
LayerNorm: Stabilizes activations for smoother training

Decoder's Secret

Masked Self-Attention: No Peeking!

During generation, the model can only attend to previous tokens, not future ones. This is enforced with a mask.

Why mask?

When generating "The cat sat on the mat", to predict "sat", the model should only see "The cat" - not the answer! This prevents information leakage during training.
