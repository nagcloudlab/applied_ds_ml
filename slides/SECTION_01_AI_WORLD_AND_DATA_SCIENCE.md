# Slide 12 - Rule-Based AI

## Human Writes the Rules

Rule-based AI uses instructions written by humans.

Example:

```text
If marks >= 35:
    result = Pass
Else:
    result = Fail
```

Good for:

- Fixed decisions
- Clear business rules
- Simple validation
- Compliance checks

Weak for:

- Complex patterns
- Changing behavior
- Ambiguous cases
- Image, speech, and language understanding

Memory line:

```text
Rule-based AI follows rules. It does not learn from examples.
```

---

# Slide 13 - Machine Learning

## Machine Learns From Examples

Machine Learning uses past data to learn patterns.

Instead of writing every rule manually, we give examples.

Example:

```text
Past student data:
attendance + assignment score + lab score -> final outcome

The model learns:
which patterns are connected with the outcome
```

Good for:

- Prediction
- Classification
- Pattern recognition
- Risk scoring
- Recommendation

Memory line:

```text
Machine Learning learns patterns from data.
```

```text

    -> layers of calculations
    -> learned pattern
    -> output
```

```text

    -> neural network
    -> "cat" or "dog"
```

```text
Neural networks learn patterns through connected layers.
```

# Slide 18 - Neurons and Layers

## How Neural Networks Are Organized

A neural network has layers:

- Input layer
- Hidden layers
- Output layer

Simple flow:

```text
Input layer
    -> hidden layer
    -> hidden layer
    -> output layer
```

Why hidden layers matter:

- Early layers learn simple patterns
- Later layers learn more complex patterns
- More layers can learn richer representations

Trainer line:

```text
Deep Learning is called deep because it uses multiple layers to learn complex patterns.
```

A feedforward neural network passes information in one direction.

Simple flow:

```text

    -> hidden layers
    -> output
```

Useful for:

- Tabular data
- Simple prediction tasks
- Learning non-linear patterns

Example:

```text

```


# Slide 25 - Transformers

## The Architecture Behind Modern LLMs

Transformers are neural network architectures designed to handle sequences efficiently.

They are powerful because they can:

- Pay attention to important parts of input
- Process long text more effectively
- Learn context
- Scale to very large datasets and models

Simple idea:

```text
Transformer = neural network architecture that learns context using attention
```

Memory line:

```text
Transformers are the backbone of modern large language models.
```

---

# Slide 26 - Attention

## How Transformers Focus on Context

Attention helps a model decide which words or parts of input matter most.

Example sentence:

```text
The student submitted the assignment because she studied well.
```

To understand `she`, the model should pay attention to:

```text
student
```

Trainer line:

```text
Attention is one reason modern language models understand context better than older sequence models.
```

# Slide 29 - Tokens and Embeddings

## How Text Becomes Numbers

Models do not directly understand text the way humans do.

Text is converted into tokens.

Example:

```text
"Data Science is useful"
    -> tokens
    -> numbers
```

Embeddings represent meaning as numbers.

Simple idea:

```text
Words with related meaning get related numeric representations.
```

Trainer line:

```text
This connects back to Data Science: even language must become data before models can learn from it.
```

# Slide 42 - Lifecycle Step 1

## Problem Understanding

Before touching data, ask:

- What question are we trying to answer?
- Who will use the answer?
- What decision will be made?
- What action will follow?
- What does success mean?
- What risks should we avoid?

Example:

```text
Weak question:
Analyze student data.

Better question:
Which patterns help us identify students who may need early academic support?
```

Memory line:

```text
A clear problem creates clear analysis.
```

- Survey forms
- Sensors
- Web platforms

- Is it reliable?

# Slide 45 - Lifecycle Step 4

## Exploratory Data Analysis

EDA means exploring data before making strong conclusions.

During EDA, we ask:

- How many rows and columns are present?
- What does each column mean?
- Which columns are numeric?
- Which columns are categorical?
- Are there missing values?
- Are there unusual values?
- What patterns are visible?
- Which relationships look important?

Trainer line:

```text
EDA is how we listen to the data before asking it to support decisions.
```

# Slide 48 - Lifecycle Step 7

Feature preparation means transforming data into a useful form for analysis or modeling.

It may include:

- Selecting useful columns
- Creating new columns
- Encoding categories
- Scaling numeric values
- Handling final missing values
- Removing identifiers
- Avoiding leakage
- Preparing data for machine learning

Memory line:

```text

```

# Slide 49 - Lifecycle Step 8

## Modeling

Modeling means using algorithms to learn patterns from data.

Examples:

- Predicting a number
- Predicting a category
- Grouping similar records
- Detecting unusual behavior
- Recommending items
- Generating text or images

Important note:

```text

```

Trainer line:

# Slide 50 - Lifecycle Step 9

## Evaluation

Evaluation checks whether the solution is useful.

For analysis:

- Are the insights correct?
- Are the charts clear?
- Are assumptions documented?
- Can stakeholders act on the findings?

For models:

- How accurate are predictions?
- What types of mistakes happen?
- Does the model work on unseen data?
- Is the model fair and explainable?

Memory line:

```text
A result is useful only if it is reliable enough for the decision.
```

# Slide 52 - Principles of Data Science

## How Professionals Think

Key principles:

- Start with the problem, not the tool
- Understand the data before modeling
- Do not trust messy data blindly
- Check assumptions
- Visualize before concluding
- Keep transformations documented
- Avoid data leakage
- Choose metrics based on the real decision
- Explain results in simple language
- Consider ethics, privacy, and fairness

